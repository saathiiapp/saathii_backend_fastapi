Authentication Endpoints

All authentication endpoints handle OTP-based verification and JWT token management.

POST /auth/request_otp
Request OTP for phone number verification

Request Body

{ "phone": "+918709996580" }

Response

{ "message": "OTP sent" }

Rate Limiting

5 requests per 15 minutes per phone number

POST /auth/resend_otp
Resend OTP with throttling (60 seconds between requests)

Request Body

{ "phone": "+918709996580" }
POST /auth/verify
Verify OTP and get authentication tokens

Request Body

{ "phone": "+918709996580", "otp": "123456" }

Response (Registered User)

{ "status": "registered", "access_token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...", "refresh_token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9..." }

Response (New User)

{ "status": "needs_registration", "registration_token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9..." }
POST /auth/register
Register new user with profile information

Request Body

{ "registration_token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...", "username": "john_doe", "sex": "male", "dob": "1990-01-01", "bio": "I love to help people", "interests": ["music", "sports"], "profile_image_url": "https://example.com/image.jpg", "preferred_language": "en", "role": "listener" }
POST /auth/refresh
Refresh access token using refresh token
POST /auth/logout
Logout user and revoke all tokens

Headers

Authorization: Bearer <access_token>

User Management Endpoints

Comprehensive user management including profiles, presence, favorites, blocking, and verification.

Profile Management

GET /users/me
Get current user's profile information
PUT /users/me
Update current user's profile
DELETE /users/me
Delete current user account

Presence & Status

GET /users/me/status
Get current user's online status and busy state
PUT /users/me/status
Update user's online/busy status
POST /users/me/heartbeat
Send heartbeat to keep user online
GET /users/{user_id}/presence
Get another user's presence status

Feed & Discovery

GET /feed/listeners
Get listeners feed with filtering options

Query Parameters

  • online_only - Show only online users
  • available_only - Show only available users (online and not busy)
  • language - Filter by preferred language
  • interests - Filter by interests (comma-separated)
  • min_rating - Minimum rating filter
  • page - Page number for pagination
  • per_page - Items per page (max 100)
GET /feed/stats
Get listener statistics for the feed

Favorites Management

POST /favorites/add
Add a listener to favorites
DELETE /favorites/remove
Remove a listener from favorites
GET /favorites
Get user's favorite listeners with pagination

Blocking System

POST /block
Block a user
DELETE /block
Unblock a user
GET /blocked
Get list of blocked users

Wallet & Earnings

GET /wallet/balance
Get wallet balance and earnings summary
GET /wallet/earnings
Get call earnings history
POST /wallet/withdraw
Request withdrawal from wallet
PUT /wallet/bank-details
Update bank details for withdrawals

Listener Verification

Call Management Endpoints

Complete call management system with coin-based billing, real-time tracking, and earnings management.

Call Operations

POST /calls/start
Start a new call with a listener

Request Body

{ "listener_id": 123, "call_type": "audio", "estimated_duration_minutes": 30 }

Response

{ "call_id": 456, "message": "Call started successfully. Maximum duration: 60 minutes", "estimated_cost": 300, "remaining_coins": 700, "call_type": "audio", "listener_id": 123, "status": "ongoing" }
POST /calls/end
End an ongoing call

Request Body

{ "call_id": 456, "reason": "completed" }
GET /calls/ongoing
Get current ongoing call for the user
POST /calls/emergency-end/{call_id}
Emergency end call (for technical issues)

Call History & Analytics

GET /calls/history
Get user's call history with pagination and filtering

Query Parameters

  • page - Page number (default: 1)
  • per_page - Items per page (default: 20, max: 100)
  • call_type - Filter by call type (audio/video)
  • status - Filter by status (ongoing/completed/dropped)
GET /calls/history/summary
Get call history summary with statistics

WebSocket Endpoints

Real-time WebSocket connections for live updates and presence tracking.

WS /ws/feed
WebSocket endpoint for real-time feed updates

Query Parameters

token - JWT access token for authentication

Connection URL

ws://localhost:8000/ws/feed?token=YOUR_ACCESS_TOKEN

Message Types

  • ping - Send ping to keep connection alive
  • subscribe - Subscribe to specific updates

Response Types

  • connection_established - Initial connection confirmation
  • pong - Response to ping
  • subscription_confirmed - Subscription confirmation
  • user_status_update - Real-time user status updates
WS /ws/presence
WebSocket endpoint for real-time presence updates

Query Parameters

token - JWT access token for authentication

Connection URL

ws://localhost:8000/ws/presence?token=YOUR_ACCESS_TOKEN

Key Features

  • Async/Await Support: Built with FastAPI for high-performance async operations
  • Database: PostgreSQL with AsyncPG for efficient database operations
  • Caching: Redis for session management, rate limiting, and real-time data
  • Authentication: JWT-based authentication with refresh token rotation
  • Real-time: WebSocket support for live updates and presence tracking
  • File Storage: AWS S3 integration for audio file uploads
  • Validation: Pydantic models for request/response validation
  • Documentation: Auto-generated OpenAPI documentation
  • Containerization: Docker support for easy deployment

API Response Codes

Success Responses

200 OK - Request successful

201 Created - Resource created successfully

Client Error Responses

400 Bad Request - Invalid request data

401 Unauthorized - Authentication required

404 Not Found - Resource not found

Server Error Responses

500 Internal Server Error - Server error occurred

Rate Limiting

  • OTP Requests: 5 requests per 15 minutes per phone number
  • OTP Resend: 1 request per 60 seconds per phone number
  • General API: No specific rate limiting (can be added as needed)

Security Features

  • JWT token-based authentication
  • Refresh token rotation for enhanced security
  • Token blacklisting on logout
  • Rate limiting on sensitive endpoints
  • Input validation using Pydantic models
  • SQL injection prevention with parameterized queries
  • File upload validation and size limits
